# Usability

## Primary journeys

1. Home page to onboarding to learning path result
2. Scheme screening to “may be eligible” result with official links
3. Pricing to Pro demo access to the gated modelling dashboard

## Design rules

- Every page has one primary CTA.
- Content is chunked into cards and short sections.
- Mobile-first layouts come first, then expand for larger screens.
- Focus states and keyboard navigation remain visible.

## Variants behind flags

- `ENABLE_ALT_HOME_HERO`: alternative hero hierarchy and CTA placement
- `ENABLE_ALT_ONBOARDING_RESULTS`: alternative results presentation on `/start`
